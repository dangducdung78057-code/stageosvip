# StageOS 唯一开发主线

## 技术基线

本仓库只保留一套生产技术栈：

- Vite + React + TypeScript
- Supabase Auth / PostgreSQL / Storage / Edge Functions
- React Three Fiber + Three.js
- Zustand
- Vitest
- Vercel

原 Next.js / Prisma 补丁只作为需求参考，不直接应用。

## 保留

- 项目、输入、方案快照、确认记录和导出数据
- 853 色传统色库与节目预设
- 舞台知识、65 条反向约束和场景规则
- 现有 3D 队形算法、男女基础 GLB、SkeletonUtils.clone
- 隐私确认与匿名学生编号

## 新增

- 统一领域 Schema
- 无 Token 可运行的本地规则方案引擎
- 2D 与 3D 共享编辑状态
- 服装 Manifest、材质槽位和资产验证
- 方案/队形/外观不可变版本
- 来源、引擎和回退状态标识
- 一条命令质量检查与 Vercel SPA 配置

## 删除或隐藏

- 把 mock 结果当正式方案的文案和状态
- 仅列出但未实现的 `/api/stageos/*` 功能入口
- 3D 人台与 15 秒视频占位卡
- 重复的色库、约束和节目映射副本
- 页面组件直接拼装数据库 JSON 的新增写法

## 暂缓

- 15 秒视频生成
- 自动下单
- 实时库存/价格承诺
- 省级及以上自动成案
- 自由骨骼动画编辑器

## 最终用户路径

1. 新建项目
2. 输入节目、人数、场地、预算和匿名学生资料
3. 本地规则预检
4. 生成服装、配色、队形、舞台、道具、风险与倒排计划
5. 2D 快速编辑
6. 3D 空间验证
7. 服装与色卡调整
8. 保存版本
9. 确认与导出

## 开发顺序

1. 领域契约与本地方案引擎
2. 统一项目工作台
3. 2D 编辑器和共享 Store
4. 3D 渲染稳定化
5. 服装库与 Manifest
6. 版本、导出、CI 与 Vercel 发布
