# StageOS AI 协作模型

## ChatGPT

### 架构

- 维护产品边界、领域模型、模块关系和阶段目标。
- 识别跨前端、数据库、鉴权、素材和部署的依赖。

### 审核

- 审核功能状态、风险、验收证据与事实一致性。
- 区分本地完成、测试通过、远端部署和生产可用。

### 规则

- 维护业务规则、隐私约束、会员边界和 AI 协作规范。
- 将模糊需求整理为可执行、可验证的任务清单。

## Codex

### 工程执行

- 在授权范围内读取、修改和验证代码与配置。
- 优先复用现有架构、类型、脚本和测试基础。

### Git

- 修改前核对分支、HEAD、状态和用户变更。
- 按授权执行精确 staging、commit、push 和只读审计。

### 测试

- 根据变更风险执行类型检查、单元测试、构建和必要的交互验证。
- 如实报告未执行、失败、环境受限或只能局部验证的项目。

## Lovable

### UI / 业务开发

- 用于页面、组件、业务流程和 Supabase 集成的快速开发。
- 产出必须回到 Git 主线审查，不能以平台预览替代仓库状态。

## v0

### 原型验证

- 用于界面结构、交互方向和视觉原型的快速验证。
- v0 导出包是代码来源之一，不携带可信 Git 历史。
- 原型进入生产主线前必须经过领域适配、鉴权、保存和测试审查。
